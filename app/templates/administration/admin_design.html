Button Usage Guidelines
  • Use Primary buttons for main actions (submit, save, create)
  • Use Secondary buttons for alternative actions (cancel, back)
  • Use Outline variants for less dominant actions
  • Use Danger buttons for destructive actions (delete, remove)
  • Use Icons to reinforce the button's action
  • Button text should be concise and action-oriented (verb + noun)
  • Maintain consistent spacing between buttons (0.5rem/8px minimum)

Implementation with uPlot:
// Create a health gauge
function createHealthGauge(element, score) {
  // Get color based on score
  let color = getHealthColor(score);

  // Create uPlot instance
  const opts = {
    width: 140,
    height: 140,
    series: [
      {},
      {
        stroke: color,
        paths: drawGauge
      }
    ]
  };

  new uPlot(opts, [[], [score]], element);

  // Add percentage display
  const valueDisplay = document.createElement('div');
  valueDisplay.className = 'health-gauge-value';
  valueDisplay.textContent = `${score}%`;
  element.appendChild(valueDisplay);
}

Chart Guidelines:

Confirmation Dialog
Confirm Deletion

Are you sure you want to delete this item? This action cannot be undone.

Best Practices:
  • Use modal-dialog-centered for confirmation dialogs
  • Keep confirmation messages concise and clear
  • Use destructive action color (danger) for delete buttons
  • Place the cancel action first, the confirm action second
  • Use verbs that match the action (Delete, not Yes)
  • Indicate the consequences of the action

Modal Usage Guidelines
  • Use modals to focus user attention on important actions
  • Keep modal content concise and focused on a single task
  • Choose appropriate modal size for the content
  • Include a clear way to dismiss the modal (close button, cancel button)
  • Use centered modals for short, important messages (confirmations)
  • Use scrollable modals for longer content (modal-dialog-scrollable)
  • Ensure modals are accessible (focus management, keyboard navigation)
  • Don't stack modals (open a modal from within another modal)

Badge Usage:
  • Use badges to highlight status, counts, or labels
  • Use appropriate colors for status (success, warning, danger)
  • Use rounded-pill for count indicators
  • Keep badge text short (1-3 characters for counts)

Notification Usage Guidelines
  • Use notifications sparingly to avoid overwhelming users
  • Match notification type to the message context (success, warning, error)

  • Use temporary toasts for non-critical information
  • Group related notifications to reduce the number of interruptions
