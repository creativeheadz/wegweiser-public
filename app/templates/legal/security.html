Security Policy

Last Updated: {{ current_year|default('2025') }}

1. Our Commitment to Security

At Wegweiser, security is fundamental to our AI-powered device intelligence platform. We implement comprehensive security measures to protect your data, systems, and privacy while providing reliable MSP services.

2. Data Protection

We employ multiple layers of security to protect your information:

  • Encryption: All data is encrypted in transit using TLS 1.3 and at rest using AES-256
  • Access Controls: Role-based access control (RBAC) with principle of least privilege
  • Authentication: Multi-factor authentication (MFA) support for enhanced account security
  • Network Security: Firewalls, intrusion detection, and network segmentation

3. Infrastructure Security

Our platform infrastructure includes:

  • Regular security assessments and penetration testing
  • Automated vulnerability scanning and patch management
  • Secure development lifecycle (SDLC) practices
  • 24/7 security monitoring and incident response
  • Backup and disaster recovery procedures

4. Agent Security

Our monitoring agents are designed with security in mind:

  • Encrypted communication channels with certificate pinning
  • Minimal system privileges and sandboxed execution
  • Regular security updates and automatic patching
  • Tamper detection and integrity verification

5. Compliance and Standards

We adhere to industry-standard security frameworks and regulations:

  • ISO 27001 security management principles
  • GDPR compliance for data protection
  • SOC 2 Type II controls implementation
  • Regular third-party security audits

6. Incident Response

In the event of a security incident, we have established procedures to:

  • Rapidly detect and contain security threats
  • Assess the scope and impact of incidents
  • Notify affected customers within 72 hours when required
  • Implement corrective measures and prevent recurrence
  • Conduct post-incident reviews and improvements

7. Responsible Disclosure

Security Researchers: We welcome responsible disclosure of security vulnerabilities. Please report security issues to our dedicated security team.

8. User Security Responsibilities

To maintain security, we recommend that you:

  • Use strong, unique passwords and enable MFA
  • Keep your systems and agents updated
  • Monitor access logs and user activity
  • Report suspicious activity immediately
  • Follow security best practices for your endpoints

9. Data Retention and Deletion

We maintain clear policies for data handling:

  • Data is retained only as long as necessary for service provision
  • Secure deletion procedures for end-of-life data
  • Customer-controlled data retention settings
  • Regular purging of unnecessary logs and temporary data

10. Security Updates

This security policy is reviewed and updated regularly to reflect:

  • Changes in our security posture and capabilities
  • Evolution of threat landscape and best practices
  • Regulatory requirements and compliance standards
  • Customer feedback and security recommendations
